## When to Use

Any time prose needs to be written: emails, Slack messages, memos, updates, proposals, briefs.

## Inputs

- What needs to be written
- Who the audience is
- The goal of the communication
- Optionally: relevant context, linked wiki pages, draft length

## Steps

1. Read `9 | Reference/comms.md` — always, without exception
2. Read `9 | Reference/identity.md` for personal context if relevant
3. Read linked wiki pages for the people and projects involved
4. Draft in the user's voice following the instructions in comms.md
5. Save to `1 | Wiki/1.6 | Exports/drafts/[title].md` with frontmatter linking to task and context

## Output

A complete draft. Never a sent message.

## Safety Rules

- Read comms.md first. Every time. No exceptions.
- Do not add preamble, sign-offs, or filler phrases the user has not approved
- Draft only — never send, post, or transmit

## What to Update Afterward

- If feedback on the draft reveals a new voice preference, update `comms.md`
